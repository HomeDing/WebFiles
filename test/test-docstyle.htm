When designing HTML areas for reading text, it's important to consider both the width of the text container and the text size to ensure readability and a comfortable reading experience.

Size

Width — A line length of 50-75 characters per line, including spaces is considered for a good reading. The standard width of the main area is set to 42em ~ 672px including some padding resulting in about 80 characters. This size also is usable on almost all mobile devices.

Text Size — The text resolution that is defined by the default font-size on a device is not changed. All font-sizes foc text are implemented by relative factors or are based on the rem units. The line-height is set to 1.2 for all text.

Colors

Unlike sizing properties, colors can be defined using CSS variables exclusively, as they don't affect element dimensions, positioning, or sizes. This approach enhances the reusability of the style toolkit and enables easy customization of the color scheme.

The color scheme is based on the light and dark modes, which can be toggled by the user or automatically based on the user's preferences.

The page-level areas have their specific coloring defined by using css variables:

--back: light-dark(hsl(220deg 2% 85%), hsl(220deg 2% 15%));
--color: light-dark(hsl(220deg 0 10%), hsl(220deg 0 90%));

--header-back: var (--back);
--header-color: var (--color);

--navbar-back: hsl(220, 80%, 30%);
--navbar-color: white;

--main-back: light-dark(hsl(220deg 0 90%), hsl(220deg 0 10%));
--main-color: var(--color);

--header-back: var (--back);
--header-color: var (--color);

--footer-back: var(--back);
--footer-color: var(--color);

--menu-back: light-dark(white, black);
--menu-color: var(--color);

These color values are defined at the :root level and can be redefined through additional page-level assignments, enabling easy style adaptations without recompiling the CSS.
